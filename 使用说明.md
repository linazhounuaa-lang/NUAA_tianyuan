# 团队季度 Workshop 网站使用说明

## 页面

- `index.html`：老师填写推荐信息。
- `reports.html`：公开展示已完成报告的专家与历史 Workshop。
- `admin.html`：负责人后台，默认口令为 `workshop2026`，可在 `app.js` 顶部修改。
- `leader.html`：领导决策页，可记录“是 / 备选 / 后续跟进 / 否”。

## 每季度操作流程

1. 负责人把 `index.html` 链接发给团队老师。
2. 老师填写推荐主题和专家信息，系统自动生成 A/B/C 初始优先级。
3. 负责人进入 `admin.html`，查看候选专家池，可修改优先级和邀请状态。
4. 点击“复制领导名单”，提交团队领导决策。
5. 领导或负责人在 `leader.html` 记录决策。
6. 在后台对确定专家点击“设为本期”，补充时间、地点、主持人、联系人等信息。
7. 在“自动文案”区域复制专家邀请微信、活动通知、主持词、新闻稿初稿。
8. 活动结束后补充参会人数、新闻稿状态、合作意向等，可上传一张报告现场图片，点击“归档本期 Workshop”。
9. 归档后的报告会自动显示在 `reports.html` 的“已报告专家”页面。

## 删除错误归档

如果有人误操作或恶意归档，负责人进入 `admin.html` 后，在“台账归档”表格中点击对应记录右侧的“删除归档”。删除后该记录也会从“已报告专家”页面移除。

## 当前版本说明

当前版本为静态网页原型，数据保存在浏览器 `localStorage` 中，适合本机演示和流程验证。若需要多人同时在线填写和长期保存数据，可使用 `schema.sql` 在 Supabase 建表，再把 `app.js` 的本地存储替换为 Supabase API。

现在代码已支持 Supabase 在线模式。填写 `config.js` 后，数据会优先保存到云端；未填写时仍使用浏览器本地缓存。详细步骤见 `线上部署说明.md`。

## 本地预览

如电脑已安装 Node.js，可在本目录运行：

```bash
node preview-server.js
```

然后访问 `http://127.0.0.1:8766/`。

也可以使用短路径访问首页：`http://127.0.0.1:8766/NUAA_tianyuan`。
